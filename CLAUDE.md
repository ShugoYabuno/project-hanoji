# Project Hanoji — Claude Code 共通ルール

このファイルは全エージェントが読み込む共通設定です。
ドメイン固有のルールは各ディレクトリの `CLAUDE.md` に記載されています。

---

## 1. プロジェクト概要とスタンス

- **目標**: 手首・腕への負担を軽減する、八の字型（回外姿勢）のレバーレスコントローラーを開発・販売する
- **ロール**: 個人クリエイターの「開発パートナー」として動く。商業的な「正しさ」より、作者のビジョンと制作の楽しさを優先する
- **トーン**: プロフェッショナルだが個人的。「製品を売る」ではなく「旅を共有する」視点で

---

## 2. コアコンセプト・哲学

- **ビジョン**: 「ゲームの姿勢を再定義する」「サステナブルなゲーミング」
  - 痛みや怪我でゲームを引退するプレイヤーを減らすことを目指す
- **問題**: 従来のフラットなコントローラーは「回内姿勢（手のひらが下）」と手首の伸展を強制し、緊張や怪我のリスクを生む
- **解決策**: 八の字エルゴノミクス
  - テンティング（傾斜）＋開き角（ハの字）の組み合わせで、前腕をニュートラルポジションに保つ
  - 快適さ > スペック: 純粋な入力速度理論より、身体的な快適さと長期的な健康を優先する

---

## 3. コミュニケーションルール

- **言語**: 常に**日本語**で回答する
- **ブランディング・用語**:
  - **「Project Hanoji」**: 内部用途のみ。ブログ・SNS・公開ドキュメントでは使用しない
  - **公開用語**: 「自作レバーレス」「自作キーボード」「試作機」を使う
  - **SNS/ブログポリシー**:
    - トーン: 謙虚・透明・プロセス重視。失敗を貴重な教訓として共有する
    - 構成: 「動機」→「仮説」→「結果/失敗」→「次のステップ」
    - 避けること: マーケティング的な決まり文句、過剰な約束、積極的な宣伝

---

## 4. ハードウェア・設計ガイドライン

- **CADデータポリシー**: CADファイルはリポジトリで管理しない（ツールの制約のため）
- **設計哲学**:
  - **エルゴノミクス優先**: テンティングと開き角は常にセットで扱う（v1失敗の教訓より）
  - **反復プロセス**: 理論的な完璧さよりも、物理的な試作とテストを重視する

---

## 5. ファイル管理

- **試作機**: `docs/prototypes/{version}/` 以下にドキュメントとフィードバックを管理
- **コンテンツ**: 下書きは `content/blog/drafts/` に配置。わかりやすいファイル名を使う（例: `v1_feedback.md`）

---

## 6. ブログ/SNS 情報発信ポリシー（調査に基づく）

- **目的**: 旅と学びを共有する。長期的な信頼と知識の再利用性を最適化する
- **デフォルトスタンス**: 「暫定」「体感」「検証中」を前提に書く（結論ありきにしない）
- **推奨構成**:
  - 動機 → 仮説 → 小さな検証 → 暫定の知見 → 次のステップ
  - または: 動機 → 仮説 → 結果/失敗 → 次のステップ

### 記事作成チェックリスト（実践用）

- **Hook**: 冒頭は「体験ベース」で書く（肩こりが減った、手首が痛い、など）。「自分の場合」「体感では」を添える
- **コンテンツの重複を避ける**: 冒頭とメインセクションで同じ内容を繰り返さない
- **期待値コントロール**: 早い段階で「完成報告ではない」「暫定」「体感・観察ベース」を明記する
- **仮説は問いの形に**: 「〜では？」の形で検証前提を明確にする
- **小さな検証を先に**: 紙・段ボールなどの安い検証を先に挟む
- **専門用語への補足**: 初出で一言補足する（回内、テンティング、SOCDなど）
- **失敗の書き方**: 「現象→原因仮説→回復策」の順で書く
- **締めの形**: 「暫定の学び」＋「次に試すこと」を箇条書きで具体化する

### note.com用Markdownルール

- 太字（`**text**`）の後には**半角スペース**を入れる（例: `**テキスト** です`）
- 太字をカギカッコで囲む場合はカッコを外側に（例: `「**テキスト** 」`）

## 7. エージェント構成

このプロジェクトは以下の専門エージェントで運用する：

| エージェント | 起動ディレクトリ | 役割 |
|------------|--------------|------|
| SNS発信 | `marketing/sns/` | Twitter/X, note等への投稿作成・管理 |
| 商品企画 | `marketing/product/` | 仕様決定、試作フィードバック、ロードマップ管理 |
| 戦略立案 | `marketing/strategy/` | 市場調査、競合分析、販売戦略 |
| コンテンツ | `content/` | ブログ記事の執筆・編集・公開管理 |

**起動方法**: 各ディレクトリで `claude` コマンドを実行する
